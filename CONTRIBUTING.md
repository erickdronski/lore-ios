# Contributing

Thank you for helping improve Lore.

1. Open an issue before substantial product or architecture work.
2. Create a focused branch from `main`.
3. Preserve privacy disclosures, curated-source boundaries, and on-device scanner behavior.
4. Add or update tests for behavior changes.
5. Run `bundle exec fastlane tests` and the relevant local checks from `README.md`.
6. Open a pull request with the problem, approach, validation, risk, and screenshots for UI changes.

Never commit signing assets, credentials, private journal data, camera frames,
or private production records. Security reports belong in the private process
described in `SECURITY.md`.
