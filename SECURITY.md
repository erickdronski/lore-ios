# Security policy

## Reporting a vulnerability

Please do not disclose vulnerabilities, credentials, private journal data,
precise location, camera content, or reproduction details in a public issue.

Use [GitHub's private vulnerability reporting flow](https://github.com/erickdronski/lore-ios/security/advisories/new).
Include the affected component, impact, reproduction steps, and any suggested
mitigation. You should receive an acknowledgement within five business days.

## Supported version

Security fixes target the current `main` branch and the current TestFlight
release candidate. Older builds are not independently supported.

## Scope

Reports involving authentication, account deletion, private notes or photos,
camera uploads, location, StoreKit, Edge Functions, or release credentials are
especially important.
